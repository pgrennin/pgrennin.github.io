The Difference Between Relative, Absolute, and Fixed Positioning in CSS

Sunday, August 17, 2014
In this post I will explain the difference relative, absolute, and fixed positioning in CSS. CSS is a programming language which allows us to add amazing style and content to webpages. Positioning in CSS is how and where objects on a webpage appear. Positioning allows us to center objects, overlay text on an image, set dimensions to objects, and much more. An important concept to understand is that all elements are made up of blocks, which can be defined by a height and width. In CSS we use the 'position' property to change how these blocks on a webpage display. There are three possible values for position in CSS:
  1. Relative
  2. Absolute
  3. Fixed

Relative Positioning

Relative positioning allows us to change the position of an element relative to its normal position. For example if we wanted to move an image 10 pixels to the right and 100px lower relative to its normal position we might use the following CSS properties and values:

#red_square{
position: relative;
left: 10px;
top: 100px;
}

In this image below, the red box would normally appear in the upper left of the container, but we moved it 10 pixels to the right and 100 pixels lower by changing it's position relative to it's normal position.

Absolute Positioning

Absolute positioning allows us to place an element anywhere on the page and set its position relative to its parent container. If there is no parent container than you can literally place the image anywhere on the page. It is similar to relative position but we are adjusting the element's position relative to the parent container rather than its normal position.

In this example below, the parent container is the light green area, and the red element is positioned relative to it.

Fixed Positioning

Fixed positioning is positioned relative to the viewport or the browser window itself and is fixed into that position. Though they're fairly rare, you most often see this implemented when the menu bar is fixed at the top of the page while you scroll down a page. In other words a fixed element is always displayed on the page even if you scroll several pages down.

Before concluding it's worth mentioning that all elements are positioned by the static property by default. Static just means the element will flow onto the page as it normally would, either horizontally (inline) or vertically (block) depending on the display type of the element.
